# 면접 예상 질문 & 답변 (QC / API 생산관리 직무)

> 이 프로젝트(충북권 대기질 SPC 모니터링)로 면접 볼 때 나올 질문과 답변 스크립트.
> 핵심 톤: 방어적("어쩔 수 없이")이 아니라 의도적("일부러 골랐다"). 대기질 용어가 아니라
> **제조/제약 QC 언어(라인·CQA·규격·CPV·OOT·4M·CAPA)로 번역**해 말한다.

## 0. 프로젝트 본 의도 (30초 엘리베이터 피치)

> "공장 실공정 데이터는 외부인이 못 구하니, **제조 공정과 통계적 구조가 동일한 실시간 실데이터**(에어코리아)를 '공정'처럼 다뤄 **SPC·6시그마·데이터 자동화 파이프라인을 무중단·무비용으로 운영**할 수 있음을 증명한 포트폴리오입니다. 측정소=라인, 오염물질=품질특성(CQA), 환경기준=규격. 대기질은 목적이 아니라 무대장치입니다."

핵심 매핑:
| 실제 직무(제조 QC) | 이 프로젝트 |
|---|---|
| 공정 라인 N개 | 측정소 5곳 |
| 품질특성(CQA) | 6오염물질 |
| 규격 USL/LSL | 대기환경기준 |
| 시간당 샘플링 | 시간당 API 호출 |
| Cp/Cpk·관리도·DMAIC | (그대로) |

## Q1. "왜 제조 데이터로 안 하고 대기질 데이터를 모았나요?"

**핵심**: 대기질이 좋아서가 아니라, 제조 공정과 통계 구조가 같으면서 외부인이 합법적으로 구할 수 있는 **유일한 실시간 실데이터**였기 때문.

> "실제 공장 공정 데이터는 기업 기밀이라 외부 지원자가 구할 수 없습니다. 그래서 제조 공정과 통계적 성질이 같은 실데이터를 찾았습니다 — 연속 발생, 다수 라인(측정소), 규격이 정해진 품질특성(오염물질)을 시간당 샘플링하는 구조죠. **가짜 공정 데이터를 만들면 정작 SPC가 잡아내야 할 결측·이상치·드리프트가 사라져 무의미**해집니다. 그래서 더미 대신 실시간·실측·지저분한 데이터로 수집 자동화부터 Cp/Cpk·관리도·가설검정까지 전 과정을 검증했습니다. **공정 데이터만 주시면 같은 파이프라인을 첫날부터 적용**할 수 있습니다."

꼬리질문 대비:
- "왜 실데이터 고집?" → QC의 존재 이유가 현실 변동·이상을 통계로 잡는 것. 실제로 복대동 통신장애(센서 결측)·시간당 누락을 멱등 수집·백필로 복구 = SCADA/MES 결측 대응과 동일.
- **톤**: "못 구해서 쓴 게 아니라, 시스템 전체를 증명하려고 일부러 골랐다."

## Q2. "그 데이터로 공정과 어떤 연관을 지을 수 있나요?"

**핵심**: 각 분석이 실제 QC 활동에 1:1 대응함을 구체적으로. 제약 용어(CPV)로 끌어올림.

> "측정소를 라인/설비로, 오염물질을 품질특성(CQA)으로 두면 제 시스템은 그대로 **CPV(Continued Process Verification, 지속적 공정검증) 시스템**이 됩니다."

| 이 프로젝트 | 제조/제약 QC |
|---|---|
| 환경기준 USL 초과 | 규격이탈 **OOS** 판정 |
| 관리도 추세 신호 | 추세이탈 **OOT** 탐지 |
| Cp/Cpk | 제약 **PPQ/밸리데이션** 필수 지표 |
| 통신장애 결측 처리 | 계측기 고장·**데이터 무결성(ALCOA+)** |
| 새벽 동반 상승 | 4M 중 **환경 공통변동** vs 특수원인 분리 |
| 측정소 간 ANOVA | **"어느 라인이 문제인가"** 검정 |
| 기상 교란변수 통제 | 불량 원인 **4M 분리** |

시나리오 예시:
> "ANOVA에서 PM2.5가 오송 측정소에서 유의하게 높게 나왔는데, 제조로 치면 '5개 라인 중 한 라인의 CQA가 통계적으로 높다'를 잡은 겁니다. 다음 단계로 기상 교란변수를 통제하려는 건, '그 라인이 진짜 설비 문제인지 원료 로트(Material)·작업조(Man) 탓인지' 분리하는 4M 분석과 같습니다. **SPC로 신호 잡고 → 가설검정으로 원인 좁히고 → 교란변수 통제로 근본원인 접근**, 이 흐름이 OOT 조사 → CAPA로 가는 GMP 절차입니다."

제약 한 방:
> "FDA 공정 밸리데이션 3단계가 CPV인데, 품질특성을 계속 수집해 관리도로 추세이탈을 감시하고 능력지수로 규격 적합성을 정량화하는 제 대시보드가 정확히 CPV 기능입니다. 데이터만 의약품 CQA로 바꾸면 됩니다."

## Q3. "대기질은 통제 가능한 공정이 아닌데, DMAIC의 Improve를 어떻게 하나요?"

(가장 날카로운 트랩 — 먼저 한계를 인정하고 통계적 깊이로 반전)

> "맞습니다, 그게 이 데이터셋의 한계입니다. 공정 파라미터를 못 바꾸니 Improve/Control은 **alert/action 한계 기반 모니터링·알림 정책**으로 매핑했습니다. 실제 공정이라면 여기서 CAPA·공정조건 변경으로 이어지겠죠. 그리고 한 발 더 — **대기질은 강한 시계열 자기상관이 있어 전통 SPC의 i.i.d. 가정을 위반하는데, 이걸 인지하고 잔차 관리도(ARIMA residual SPC)로 보정하는 설계까지 반영**했습니다. 한계를 아는 데서 그치지 않고 통계적으로 다룰 줄 안다는 걸 보여주고 싶었습니다."

## Q4. "기상 같은 외부 요인은 고려했나요?"

**핵심**: 기상이 1순위 교란변수임을 인지·통제하려 했다 = 4M 분리 역량.

> "네, 기상은 이 분석의 1순위 교란변수입니다. 측정소 5곳 상관이 0.71~0.90으로 매우 높아 광역 공통인자(혼합고·풍계·정체)가 지배적이고, 새벽 동반 피크는 야간 역전층 일변화로 설명됩니다. 그래서 **풍향·풍속·시간대를 통제하지 않은 t-test 결과는 '탐색적·연관' 수준으로 톤다운**했고, 기상청 ASOS 결합으로 통제하는 걸 다음 단계로 설계했습니다. 제조로 치면 '설비 차이로 보였던 게 사실은 환경(4M) 탓일 수 있다'를 분리하는 작업입니다."

---

## Q5. "통계적으로 가장 신경 쓴 부분은?"

> "전통 SPC 관리도가 관측치 독립(i.i.d.)을 가정하는데, 대기질·연속공정 데이터는 강한 자기상관이 있습니다. MR(n=2) 기반 σ 추정은 이때 σ를 과소추정해 관리한계가 좁아지고 **거짓경보가 폭증**합니다. 이걸 인지해서, 자기상관을 ACF로 진단하고 잔차 관리도로 보정한 뒤 **거짓경보율 Before/After를 비교**하는 걸 핵심 과제로 잡았습니다. SPC를 '구현'하는 것과 '실무에 신뢰성 있게 쓰는 것'의 차이라고 생각합니다."

## Q6. "무중단·무비용 운영은 어떻게?"

> "GitHub Actions가 시간당 자동 수집해 SQLite에 INSERT OR IGNORE로 중복 없이 적재하고, 결과 DB를 레포에 커밋하면 Streamlit Cloud가 자동 재배포됩니다. 사용자 PC가 꺼져도 GitHub 서버에서 돌아 무중단이고, public repo Actions + Streamlit Community Tier라 무비용입니다. **GitHub Actions cron이 best-effort라 누락이 잦은 한계**가 있어, 직전 시간대 갭을 감지해 기간조회로 자동 백필하는 self-healing 구조를 다음 단계로 설계했습니다."

---

## Q7. "테스트/품질관리는?"

> "pytest 93건으로 변환·마스킹·DB 제약·SPC 계산·가설검정을 회귀 검증합니다. 인증키는 .env/Secret으로 분리하고 로그에서 마스킹합니다. 매일 자동 루프가 pytest + 누적통계 + Cpk 매트릭스 + 마일스톤을 리포트로 남깁니다."

## 자주 나올 추가 질문 (짧은 탄착점)

- **"Cp와 Cpk 차이?"** → Cp는 분포 폭 대 규격 폭(위치 무관), Cpk는 중심 치우침까지 반영. Cp 좋아도 Cpk 나쁘면 중심이 한쪽으로 쏠린 것.
- **"EWMA vs CUSUM 언제?"** → 둘 다 작은 평균이동(0.5~1.5σ)에 민감. 단발 튐은 I-Chart, 지속적 미세이동은 EWMA/CUSUM이 빨리 탐지.
- **"왜 SQLite?"** → 단일 파일이라 레포 영속화·배포가 쉬움. 데이터가 수 MB 넘으면 Turso(libSQL, SQLite 호환)나 Postgres로 이전 계획.
- **"표본 30개 기준 이유?"** → 중심극한정리 관례 임계 + Cp/Cpk 추정 안정성. 미만이면 계산을 거부하도록 가드.
- **"이 프로젝트의 한계?"** → ①공정 조작 불가(Improve 제약) ②자기상관 미보정분 잔존 ③1개월·단일계절이라 일반화 한계. **한계를 명확히 말하는 것 자체가 분석가 성숙도.**
