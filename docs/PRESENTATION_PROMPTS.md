# 발표자료 제작 프롬프트 모음 (Claude)

> 출처: "파워포인트는 이제 그만 — Claude로 120초 발표자료" 가이드.
> 용도: 본 프로젝트의 포트폴리오 발표 덱(`scripts/generate_portfolio_ppt.py`,
> `reports/portfolio/*.pptx`) 보강·재구성 시 활용. `[주제]`에는
> "충북권 산업단지 대기질 SPC 모니터링 — QC/생산관리 직무 포트폴리오"를 넣으면 된다.

## PROMPT 01 — 발표자료 설계
당신은 TED 강연, Fortune 500 기업 보고서, 투자 유치 덱을 제작해 온 세계 최고 수준의 프레젠테이션 컨설턴트입니다. [주제]에 대한 완전한 발표자료 설계안을 만들어 주세요. 먼저 아래 내용을 분석하세요: 발표의 핵심 목표, 청중의 배경지식 수준, 청중의 관심사와 기대, 예상 질문 및 반론, 발표 후 청중이 기억해야 할 핵심 메시지, 발표 후 청중이 취해야 할 행동. 그 후 아래 형식으로 작성하세요: 발표 목표, 청중 분석, 핵심 메시지, 권장 슬라이드 수, 전체 스토리라인, 슬라이드 흐름, 예상 질문 및 대응 포인트, 최종 Takeaway. 구조는 논리적이고 설득력 있으며 청중의 몰입도를 유지할 수 있도록 설계하세요.

## PROMPT 02 — 슬라이드 구조 설계
[주제]에 대한 발표자료의 슬라이드별 구조를 설계해 주세요. 각 슬라이드마다 아래 항목을 포함하세요: 슬라이드 번호, 슬라이드 제목, 해당 슬라이드의 핵심 메시지, 슬라이드 목적, 포함해야 할 주요 내용, 다음 슬라이드와 연결되는 흐름. 발표 전체가 하나의 스토리처럼 자연스럽게 이어지도록 구성하고, 불필요하거나 중복되는 슬라이드는 제거하세요.

## PROMPT 03 — 스토리텔링 기반 발표자료
[주제]를 매력적인 발표자료로 만들어 주세요. 다음 구조를 활용하세요: 훅(Hook) → 문제 제기 → 인사이트 → 해결책 → 핵심 요약. 각 단계별로: 훅은 청중이 멈추게 만드는 충격적 사실·질문·통계, 문제 제기는 청중이 공감하는 현재의 고통·불편, 인사이트는 대부분이 모르는 핵심 관점, 해결책은 구체적이고 실행 가능한 방법, 요약은 청중이 기억해야 할 딱 하나의 메시지. 전문성과 정보 전달력을 유지하면서도 청중이 몰입할 수 있도록 구성해 주세요.

## PROMPT 04 — 발표 스크립트
아래 발표자료를 기반으로 실제 발표할 수 있는 스크립트를 작성해 주세요. [슬라이드 내용 또는 발표 주제를 여기에 붙여넣기]. 스크립트 작성 조건: 자연스러운 구어체(문어체·격식체 아님), 각 슬라이드 전환 시 연결 멘트 포함, 강조할 부분은 [강조] 표시, 예상 소요 시간 표시(예: [약 1분30초]), 청중 참여 유도 포인트 표시. 발표 정보: 총 발표 시간 [N분], 청중 [청중 설명], 발표 목적 [설득/보고/교육 중 선택]. 읽었을 때 자연스럽게 말할 수 있는 스크립트로 만들어 주세요. '이제 다음 슬라이드를 보시겠습니다' 같은 어색한 표현은 제거해 주세요.

## PROMPT 05 — 전체 슬라이드 내용
[주제]에 대한 발표자료의 모든 슬라이드 내용을 작성해 주세요. 각 슬라이드마다 발표용으로 적합한 핵심 불릿포인트를 제공하고, 명확하고 전문적이며 이해하기 쉽게 작성해 주세요. 작성 형식: ## 슬라이드 N: [제목], 핵심 메시지: [한 줄], • 불릿포인트 1, • 불릿포인트 2, • 불릿포인트 3, [강조 요소 또는 시각화 제안]. 각 슬라이드는 하나의 핵심 아이디어만 전달하도록 작성해 주세요. 청중: [청중 설명], 총 슬라이드 수: [N장]. 바로 PPT에 복붙할 수 있는 형태로 작성해 주세요.

## PROMPT 06 — 압축·가독성 개선
다음 발표자료 내용을 검토하고 슬라이드에 적합한 형태로 다시 작성해 주세요. 불필요한 텍스트를 줄이고, 핵심 메시지를 강조하며, 가독성을 높여 주세요. 각 슬라이드는 하나의 핵심 아이디어만 전달하도록 수정해 주세요. 개선 기준: 슬라이드에 문장 금지 — 키워드·구 형태로, 핵심 메시지를 첫 불릿으로 강조, 3~5개 불릿 이내, 숫자·데이터는 시각화 형태로 제안. 내용: [내용 붙여넣기]. 원본의 핵심 메시지는 100% 유지하되 표현 방식을 슬라이드에 최적화해 주세요.

## 본 포트폴리오에 적용 시 권장 흐름
1. **PROMPT 01**로 전체 설계(목표·청중·스토리라인) — 청중 = "SK하이닉스/제약 QC·생산관리 면접관".
2. **PROMPT 03**(스토리텔링)으로 훅 잡기 — 예: "측정소 5곳 Cpk가 전부 1 미만이었다. 그런데 진짜 문제는 위치가 아니었다."
3. **PROMPT 05**로 슬라이드별 불릿 작성 → `docs/INSIGHTS.md`·DMAIC 결과를 근거로.
4. **PROMPT 06**으로 압축(슬라이드당 1메시지, 3~5불릿).
5. **PROMPT 04**로 발표 스크립트(시간 배분·구어체).
> 핵심 셀링 포인트(스토리 축): 통계적 유의 ≠ 실질적 중요 / 통제가능 vs 통제불가 변동 분리 / 자기상관 보정(거짓경보 48%→2%) / 무중단·무비용 자동 운영.
